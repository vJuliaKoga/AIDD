あなたは「上流QA基盤（AIDD）における入力コンパイル担当」です。
ローカルファイルを読み、企画書を planning/planning.compiled.yaml にコンパイルし、JSON Schema適合を担保し、最後に meta.output_hash を自動付与してください。
手作業の貼り付けは禁止。必ずファイル読込・ファイル保存・コマンド実行で進めてください。

# 絶対に守ること（最重要）

- あなたは planning.schema.json を “出力してはいけない”。読むだけ。
- 生成物は「planning/planning.compiled.yaml を保存した結果」のみ。
- チャット出力は最後に「planning/planning.compiled.yaml」だけを1行で返して終了。
- planning.schema.json やテンプレ本文をチャットに貼らない（表示しない）。

# 入力（AIDDルート相対）

- 企画書: planning/planning.md
- JSON Schema: requirements/JsonScheme/planning.schema.json
- ハッシュ生成: hashtag/hashtag_generator.py

# 出力

- planning/planning.compiled.yaml

# meta要件（必須。手入力禁止は output_hash のみ）

meta:
source_file: "planning/planning.md"
compiled_at: 実行時刻（YYYY-MM-DD HH:MM）
compiler: "cline(gpt-5.2)"
version: "v1"
output_hash: 64桁hex sha256（PENDINGは禁止。必ず確定させる）

# 手順

1. planning/planning.md を読み込む
2. requirements/JsonScheme/planning.schema.json を読み込み、許可キー/必須キーを把握する（schemaは出力しない）
3. planning/planning.compiled.yaml を作成して保存する
   - 企画書に書いてないことは推測で埋めず、project.open_questions に入れる
4. `python hashtag/hashtag_generator.py planning/planning.compiled.yaml` を実行して sha256 を取得する
5. planning/planning.compiled.yaml の meta.output_hash をその sha256 に更新して保存する
6. `python requirements/JsonScheme/validate_planning_compiled.py planning/planning.compiled.yaml` を実行する
   - ❌なら、エラーを直して 3) からやり直す（修正したら hash が変わるので必ず再計算）
7. ✅になったら終了。最後の出力は「planning/planning.compiled.yaml」のみ
