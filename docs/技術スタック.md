## 3. データフロー

```
[ユーザー入力]
    │
    ▼
[フロントエンド検証]
    ├── リアルタイム曖昧語チェック（Zod + 正規表現）
    └── 構文チェック（Zod スキーマ）
    │
    ▼
[REST API / Express]
    │
    ▼
[品質ゲート実行]
    ├── G1: 曖昧語チェック（正規表現）
    ├── G2: 検証可能性チェック（チェックリスト照合）
    ├── G3: JSON Schema検証（ajv）
    ├── G4: Deep Eval評価（OpenAI API経由）
    └── G5: トレーサビリティチェック（DB照合）
    │
    ├── PASS → [SQLite保存] → [Allure Report更新]
    └── FAIL → [エラー詳細返却] → [FEフィードバック表示]
```

## 4. データベース設計（SQLite）

### 主要テーブル

```sql
-- 工程
CREATE TABLE phases (
  id         TEXT PRIMARY KEY,  -- REQ, BAS, DET
  title      TEXT NOT NULL,
  status     TEXT DEFAULT 'pending', -- active|completed|pending
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- タスクチケット
CREATE TABLE tasks (
  id          TEXT PRIMARY KEY,  -- REQ-T001 等
  phase_id    TEXT REFERENCES phases(id),
  title       TEXT NOT NULL,
  purpose     TEXT,
  status      TEXT DEFAULT 'ToDo', -- ToDo|InProgress|Done|Abort
  priority    TEXT DEFAULT 'medium',
  created_at  TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updated_at  TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- Abortログ
CREATE TABLE abort_logs (
  id                TEXT PRIMARY KEY,
  task_id           TEXT REFERENCES tasks(id),
  aborted_by        TEXT NOT NULL,
  aborted_at        TIMESTAMP NOT NULL,
  reason_category   TEXT NOT NULL, -- scopeout|prerequisite_missing|delegated|not_applicable
  reason_detail     TEXT NOT NULL CHECK(LENGTH(reason_detail) >= 50),
  alternative_action TEXT,
  approved_by       TEXT,
  created_at        TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 判断ログ
CREATE TABLE decision_logs (
  id                TEXT PRIMARY KEY,
  task_id           TEXT REFERENCES tasks(id),
  decided_by        TEXT NOT NULL,
  decided_at        TIMESTAMP NOT NULL,
  decision_summary  TEXT NOT NULL CHECK(LENGTH(decision_summary) >= 50),
  rationale         TEXT NOT NULL,
  evidence          TEXT,
  risks_mitigation  TEXT,
  importance        TEXT DEFAULT 'Medium', -- High|Medium|Low
  created_at        TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- トレーサビリティ
CREATE TABLE traceability (
  from_id   TEXT NOT NULL,
  to_id     TEXT NOT NULL,
  relation  TEXT NOT NULL, -- derived_from|traces_to|prompts_used
  PRIMARY KEY (from_id, to_id, relation)
);

-- 承認者
CREATE TABLE approvers (
  id         INTEGER PRIMARY KEY,
  name       TEXT NOT NULL,
  role       TEXT NOT NULL, -- admin|qalead|architect
  email      TEXT,
  active     BOOLEAN DEFAULT 1,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 品質評価結果
CREATE TABLE quality_results (
  id           TEXT PRIMARY KEY,
  task_id      TEXT REFERENCES tasks(id),
  evaluated_at TIMESTAMP NOT NULL,
  g1_pass      BOOLEAN,
  g2_pass      BOOLEAN,
  g3_pass      BOOLEAN,
  g4_score     REAL,
  g4_pass      BOOLEAN,
  g5_coverage  REAL,
  g5_pass      BOOLEAN,
  overall_pass BOOLEAN,
  detail_json  TEXT -- エラー詳細JSON
);

-- スナップショット
CREATE TABLE snapshots (
  id            TEXT PRIMARY KEY, -- SNAP-YYYYMMDD-HHMMSS
  created_at    TIMESTAMP NOT NULL,
  created_by    TEXT NOT NULL,
  project_name  TEXT NOT NULL,
  summary_json  TEXT NOT NULL, -- 品質メトリクスサマリー
  export_path   TEXT
);
```

## 5. 品質保証ツール設定

### Deep Eval（閾値設定）

```python
# deepeval.config.py
threshold = 0.8
alert_threshold = 0.75

metrics = [
    {"name": "Contextual Relevancy",  "weight": 0.25},
    {"name": "Faithfulness",          "weight": 0.25},
    {"name": "Contextual Precision",  "weight": 0.25},
    {"name": "Answer Relevancy",      "weight": 0.25},
]
```

### Promptfoo（評価設定）

```yaml
# promptfoo.config.yaml
scoring:
  threshold: 0.8
  metrics:
    - accuracy
    - consistency
    - completeness
```

### GitHub Actions CI

```yaml
# .github/workflows/ci.yml のトリガー条件
on:
  push:
    branches: [main, develop]
  pull_request:
    branches: [main]

# 品質ゲート自動チェック項目
gates:
  - ambiguous_word_check    # G1
  - schema_validation       # G3
  - traceability_check      # G5
  - deepeval_score          # G4（PRのみ）
```

## 6. 開発環境セットアップ

## 7. 制約と注意事項

| 項目 | 内容 |
|------|------|
| AI API | OpenAI API のみ。他プロバイダーへの変更は設計変更が必要 |
| OS | Windows 11 Home での動作確認。Linux/macOSは未対応 |
| ブラウザ | Edge・Chrome のみ。Firefox・Safari は対象外 |
| モバイル | 非対応。レスポンシブ設計は行わない |
| データ保存 | ローカルSQLite。クラウド同期は将来対応 |
| 同時ユーザー | 1人（個人プロジェクト規模）。マルチユーザー対応は将来課題 |
