## 3. 画面別仕様

### SCR-001: ダッシュボード

**目的**: プロジェクト全体の品質状態を3クリック以内で把握できる起点

**表示要素**:
- 工程カード一覧（L1: 工程選択）
  - 工程名、完了率（%）、タスク数
  - ステータス色分け（active/completed/pending）
- 品質サマリーパネル
  - Deep Eval Score（最新値 + 目標値 0.8）
  - 曖昧語検出数（目標: 0件）
  - トレーサビリティカバレッジ（目標: 100%）
- スコア推移グラフ（直近30日間の折れ線グラフ）
- Abort分析（件数・カテゴリ分布円グラフ）

**操作**:
- 工程カードクリック → SCR-002へ遷移
- 判断ログリンク → SCR-007へ遷移
- エクスポートボタン → SCR-008へ遷移

---

### SCR-002: 工程詳細

**目的**: 選択した工程内のタスク一覧を確認・操作する

**表示要素**:
- 工程名・進捗バー（Done数/全タスク数）
- タスクカード一覧（L2: タスク）
  - タスクID、タスク名、ステータスバッジ
  - 優先度インジケーター（high/medium/low）
  - ステータス左ボーダー色（ToDo:gray / InProgress:blue / Done:green / Abort:orange）
- フィルター（ステータス別・優先度別）
- トレーサビリティ表示ボタン

**操作**:
- タスクカードクリック → SCR-003（オーバーレイ）を開く
- トレーサビリティボタン → SCR-005へ遷移

---

### SCR-003: タスク詳細オーバーレイ

**目的**: タスクの詳細情報・チェックリスト・テンプレート・判断記録を一箇所で操作する

**表示要素**:
- タブ構成
  - `概要` — タスク目的・受入基準・関連IDリスト
  - `チェックリスト` — CHKアイテム（完了/未完了トグル）
  - `テンプレート` — TPLコンテンツ（YAML表示・コピー機能）
  - `プロンプト` — PRMコンテンツ（コピー機能）
  - `判断ログ` — 過去の判断記録・新規記録フォーム
- アクションボタン
  - `品質ゲート実行` → SCR-004へ
  - `Doneにする`（チェックリスト完了時のみ有効）
  - `Abortする` → SCR-006へ
- リアルタイム曖昧語警告（テキスト入力欄に常時監視）

**レイアウト**:
- position: fixed / right側スライドイン
- width: 60vw (max: 800px)
- animation: slide-in-right 0.3s
- 背景: 半透明オーバーレイ（クリックで閉じない）
- 閉じる: ×ボタン または ESCキー

---

### SCR-004: 品質ゲート結果

**目的**: G1〜G5の評価結果を詳細表示し、不合格箇所の修正を促す

**表示要素**:
- ゲート別結果カード（G1〜G5）
  - PASS/FAIL バッジ
  - 詳細エラー内容（FAIL時）
  - 修正ガイドリンク
- 総合判定（GATES: PASS / GATES: FAIL）
- 再実行ボタン

**警告メッセージ例**:
| 検知 | メッセージ |
|------|-----------|
| 曖昧語検出 | 「"適切に" は曖昧な表現です。"3秒以内に" 等の具体的な表現に置換してください」 |
| 必須項目未入力でDone遷移 | 「チェックリストに未完了項目があります」 |
| Abort理由不足 | 「理由が50文字未満です（現在: XX文字）」 |
| トレーサビリティ欠落 | 「未紐付けの要件があります: REQ-FNC-XXX」 |
| 重複ID | 「このIDは既に使用されています: XXX-XXX-XXX-001」 |

---

### SCR-005: トレーサビリティマトリクス

**目的**: 要件→設計→テストの紐付け状況をヒートマップで可視化する

**表示要素**:
- ヒートマップ（行: 要件ID / 列: 設計ID・テストID）
  - 紐付けあり: 緑
  - 紐付けなし: 赤（警告）
- React Flowによるフロー図ビュー
  - ノード: 工程（Done/Abort含む）
  - エッジ: 実施順序（実線）/ Abort分岐（点線）

---

### SCR-006: Abort記録フォーム

**目的**: Abort理由を構造化して記録し、後から検索・参照できる状態にする

**必須入力項目**:
- カテゴリ（選択式）
  - `scopeout` — プロジェクトスコープ外
  - `prerequisite_missing` — 前提条件未確定
  - `delegated` — 委託元で実施済
  - `not_applicable` — 今回適用不要
- 詳細理由（テキストエリア、50文字以上必須）
- 代替措置（テキストエリア）
- 承認者（プルダウン、DBの approvers テーブルから取得）

**バリデーション**:
- 詳細理由が50文字未満の場合: 送信ボタン無効化 + 文字数カウンター赤表示
- カテゴリ未選択の場合: 送信ボタン無効化

---

### SCR-007: 判断ログ一覧

**表示要素**:
- 判断ログカード一覧（新しい順）
  - decidedby / decidedat / decisionsummary（先頭100文字）
  - 重要度バッジ（High/Medium/Low）
- 検索（タスクID・担当者名・日付範囲）
- 詳細展開（クリックで rationale・evidence・risks を表示）

---

### SCR-008: エクスポート

**選択項目**:
- 対象（プロジェクト全体 / 工程指定 / タスク指定）
- 形式（JSON / YAML / PDF）
- 含む情報（チェックリスト / 判断ログ / Abortログ / 品質メトリクス）

---

### SCR-009: 設定

**セクション**:
- 承認者管理（CRUD）
- 品質ゲート閾値（Deep Eval閾値 / アラート閾値）
- バックアップ設定（スケジュール / 保存先）
- モード切替（初心者モード / 熟練者モード）

## 5. インタラクションルール

- **3クリック原則**: 工程選択 → タスク選択 → 詳細確認の3ステップで完結
- **曖昧語監視**: テキスト入力欄はすべてリアルタイム監視対象
- **進むには品質ゲートを通過**: Done遷移にはチェックリスト完了が必須
- **Abortは正式な判断**: 理由なきAbortは送信不可
- **ESCキー**: オーバーレイを閉じる（データ未保存の場合は確認ダイアログ）
