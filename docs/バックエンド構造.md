## 1. 設計方針

- **Node.js + Express** でローカル完結のREST APIを提供する
- **SQLite** をデータストアとして使用。ファイル単一・ポータブル
- **品質ゲート（G1〜G5）はバックエンドで実行**し、フロントエンドはUIのみに集中する
- **OpenAI API** は Deep Eval（G4）でのみ呼び出す。呼び出しコストを可視化するためログを保存する
- すべてのレスポンスは共通エラー形式を持つ

## 4. 品質ゲート実装

### 4.1 ゲートオーケストレーター

```typescript
// gates/index.ts
interface GateResult {
  gate: 'G1' | 'G2' | 'G3' | 'G4' | 'G5';
  pass: boolean;
  score?: number;
  errors: GateError[];
}

interface RunGatesResult {
  taskId: string;
  evaluatedAt: string;
  results: GateResult[];
  overallPass: boolean;       // 全ゲートPASSのみtrue
  summary: string;            // 'GATES: PASS' | 'GATES: FAIL'
}

async function runAllGates(taskId: string): Promise<RunGatesResult> {
  const task = await taskRepository.findById(taskId);
  const [g1, g2, g3, g4, g5] = await Promise.all([
    runG1(task),
    runG2(task),
    runG3(task),
    runG4(task),   // OpenAI API 呼び出しのみ非同期最重視
    runG5(task),
  ]);
  const results = [g1, g2, g3, g4, g5];
  const overallPass = results.every(r => r.pass);
  await qualityRepository.save({ taskId, results, overallPass });
  return { taskId, evaluatedAt: new Date().toISOString(), results, overallPass,
           summary: overallPass ? 'GATES: PASS' : 'GATES: FAIL' };
}
```

### 4.2 G1: 曖昧語チェック

```typescript
// gates/g1-ambiguous.ts
const AMBIGUOUS_PATTERNS = [
  /適切に/g, /十分に/g, /柔軟に/g, /簡単に/g, /最適に/g,
  /基本的に/g, /一般的に/g, /必要に応じて/g, /適宜/g,
  /なるべく/g, /できるだけ/g, /スムーズに/g, /うまく/g,
];

function runG1(task: Task): GateResult {
  const text = [task.purpose, task.acceptance_criteria].join('\n');
  const detected = AMBIGUOUS_PATTERNS.flatMap(pattern => {
    const matches = text.match(pattern) ?? [];
    return matches.map(m => ({ word: m, pattern: pattern.source }));
  });
  return {
    gate: 'G1',
    pass: detected.length === 0,
    errors: detected.map(d => ({
      type: 'ambiguous_word',
      message: `曖昧語「${d.word}」を検出。定量的な表現に置換してください`,
    })),
  };
}
```

### 4.3 G3: JSON Schema検証

```typescript
// gates/g3-schema.ts
import Ajv from 'ajv';
import taskSchema from '../../shared/schemas/task.schema.json';

const ajv = new Ajv({ allErrors: true });
const validate = ajv.compile(taskSchema);

function runG3(task: Task): GateResult {
  const valid = validate(task);
  return {
    gate: 'G3',
    pass: !!valid,
    errors: (validate.errors ?? []).map(e => ({
      type: 'schema_violation',
      message: `${e.instancePath}: ${e.message}`,
    })),
  };
}
```

### 4.4 G4: Deep Eval評価

```typescript
// gates/g4-deepeval.ts
// OpenAI API を使ってコンテンツを評価する
const DEEPEVAL_THRESHOLD = 0.8;
const ALERT_THRESHOLD = 0.75;

async function runG4(task: Task): Promise<GateResult> {
  const score = await callOpenAIEvaluation(task);
  // スコアログ保存（コスト可視化のため）
  await logApiCall({ taskId: task.id, score, model: 'gpt-4o' });
  return {
    gate: 'G4',
    pass: score >= DEEPEVAL_THRESHOLD,
    score,
    errors: score < DEEPEVAL_THRESHOLD ? [{
      type: 'score_below_threshold',
      message: `Deep Eval スコア ${score.toFixed(2)} < 閾値 ${DEEPEVAL_THRESHOLD}`,
    }] : [],
  };
}
```

### 4.5 G5: トレーサビリティチェック

```typescript
// gates/g5-traceability.ts
async function runG5(task: Task): Promise<GateResult> {
  const traces = await traceabilityRepository.findByFromId(task.id);
  const hasDerivedFrom = traces.some(t => t.relation === 'derived_from');
  const hasTracesTo = traces.some(t => t.relation === 'traces_to');
  const errors: GateError[] = [];
  if (!hasDerivedFrom) errors.push({
    type: 'missing_derived_from',
    message: '企画書へのトレーサビリティ（derived_from）が設定されていません',
  });
  if (!hasTracesTo) errors.push({
    type: 'missing_traces_to',
    message: '設計・テストへのトレーサビリティ（traces_to）が設定されていません',
  });
  const coverage = errors.length === 0 ? 100 : (hasDerivedFrom || hasTracesTo ? 50 : 0);
  return { gate: 'G5', pass: errors.length === 0, score: coverage, errors };
}
```

## 5. データ整合性チェック（CI/CD）

以下はバックエンド起動時またはCI/CDで実行されるデータ品質チェック:

| チェック名 | 内容 | 失敗時の挙動 |
|-----------|------|------------|
| Abort Log Completeness | 全AbortタスクにAbortログが存在するか | ビルド警告 |
| Decision Log Min Length | 判断ログが50文字以上か | ビルド警告 |
| ID Uniqueness | 全IDが一意か | ビルド失敗 |
| Traceability Integrity | 参照IDが実在するか | ビルド失敗 |
| Date Format Validation | 日時がISO8601形式か | ビルド警告 |

## 6. バックアップ設計

```typescript
// services/backupService.ts

// スケジュール: 毎日深夜2時（node-cron）
// '0 2 * * *'

async function runDailyBackup(): Promise<void> {
  const timestamp = new Date().toISOString().replace(/[:.]/g, '-');

  // 1. SQLiteファイルコピー
  await copyFile(DB_PATH, `${BACKUP_DIR}/db-${timestamp}.sqlite`);

  // 2. JSONエクスポート（最新状態の全データ）
  const allData = await exportAllData();
  await writeFile(`${BACKUP_DIR}/export-${timestamp}.json`,
                  JSON.stringify(allData, null, 2));

  // 3. 30日超の古いバックアップを削除
  await cleanupOldBackups(30);
}
```

```typescript
interface ApiError {
  error: {
    code: string;           // 'VALIDATION_ERROR' | 'NOT_FOUND' | 'GATE_FAIL' ...

    details?: unknown;
  };
}
```

### 7.2 HTTPステータスコード方針

| 状況 | ステータス |
|------|-----------|
| 正常 | 200 / 201 |
| バリデーションエラー | 400 |
| リソース未存在 | 404 |
| ビジネスルール違反（Abort理由不足等） | 422 |
| OpenAI API エラー | 502 |
| 内部エラー | 500 |
